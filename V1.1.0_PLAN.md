#### 1.3 实时通知核心功能
**目标**: 实现低延迟的实时通知

**文件修改**: 
- `server/websocket.js`
- `server/server.js`

**实现要点**:
```javascript
// WebSocket连接管理
const wsServer = new WebSocket.Server({ server });
const connections = new Map();

wsServer.on('connection', (ws, req) => {
  const token = extractToken(req);
  const user = verifyToken(token);
  connections.set(user.id, ws);
});

// 事件广播
const broadcastNotification = (event, data, userIds) => {
  userIds.forEach(userId => {
    const ws = connections.get(userId);
    if (ws && ws.readyState === WebSocket.OPEN) {
      ws.send(JSON.stringify({ event, data }));
    }
  });
};
```

## 🚨 风险评估

### 高风险
- **支付安全风险**: 需要严格验证支付回调，防止伪造支付
- **打印机兼容性**: 不同品牌打印机可能存在兼容性问题
- **WebSocket稳定性**: 高并发场景下WebSocket连接可能不稳定

### 中风险
- **证书管理**: 支付证书的安全存储和管理
- **网络依赖**: 打印和支付功能对网络稳定性有较高要求
- **浏览器兼容性**: WebSocket在老旧浏览器中可能不支持

### 低风险
- **日志文件增长**: WebSocket和支付日志可能导致磁盘空间占用
- **内存泄漏**: WebSocket连接未正确关闭可能导致内存泄漏
- **配置复杂性**: 支付和打印配置项较多，增加使用难度
